Something else besides memory protection that I remember the Amiga didn’t do was clean up a programme’s resources if it crashed, or if it wasn’t programmed properly to close all its handles. So if a programme died unexpectedly any memory it allocated or files that it opened would still be in use.

延伸問答

Amiga 1000 的「全搶佔式多工處理」與其他系統相比有何特別之處？

「全搶佔式多工處理」允許作業系統完全控制 CPU 的時間分配，確保每個程式都能公平運行，即使一個程式卡住也不會影響其他程式。這與當時許多作業系統（如 Windows 3.0/3.1 和 Apple System 7）採用的協同式多工處理不同，後者需要程式主動放棄 CPU 控制權，更容易導致系統不穩定。

既然 Amiga 1000 技術如此先進且價格合理，為何它未能主導市場？

Amiga 1000 未能主導市場主要歸咎於 Commodore 公司糟糕的行銷策略和領導層。他們未能有效地向大眾傳達 Amiga 的獨特優勢和未來願景，也缺乏像 Apple 那樣吸引人的產品敘事。此外，公司內部管理不善，領導人更專注於從公司中謀取利益而非長期發展。

後來的個人電腦或作業系統吸收了 Amiga 的哪些創新或改進？

Amiga 1000 的全搶佔式多工處理能力直到 1995 年 Windows 95 推出後才被主流 PC 作業系統普遍採納。文章指出，作者在使用 Windows PC 時，甚至到 1994 年仍感覺其穩定性不如 Amiga，特別是在同時進行多個任務時。Amiga 在圖形和聲音方面的早期領先也為未來多媒體PC的發展奠定了基礎。

This post muses about the malleable computing capabilities provided by Emacs, showing by concrete example (fj.el) how new behavior can be created leveraging both code and program reuse in an improvised fashion. With measured expectations (requirements, feature set, audience), malleable techniques allow for building tools that would otherwise be infeasible without it.

延伸問答

為什麼作者偏好將GitHub議題複製到Org Agenda，而不是直接在GitHub上處理所有事務？

作者將GitHub議題複製到Org Agenda，是為了將其作為個人筆記的草稿區和準備回覆的暫存區。Org mode提供了更強大的表達語法和組織功能，讓作者能夠更靈活地思考和構思內容，這是單純的GitHub議題介面所無法提供的。

在Emacs的背景下，「可塑計算」（Malleable Computing）具體指的是什麼？它如何協助實現這類自動化？

在Emacs中，「可塑計算」指的是其基於Lisp的動態環境，允許用戶在運行時修改、擴展和評估程式碼。這項特性使得開發者能夠迅速建立原型、即時測試，並將內建的Elisp程式碼與外部工具（如GitHub CLI `gh`）無縫整合，從而快速地為個人化需求打造自動化解決方案。

除了同步GitHub議題外，Emacs的可塑計算能力還能應用於哪些其他類型的任務或工作流程？

Emacs的可塑計算能力適用於各種需要高度客製化和自動化的任務。例如，可以用來建立個人化的檔案管理工具、客製化專案追蹤系統、自動化資料轉換流程，或是將多個網路服務（如電子郵件、日曆、RSS閱讀器）整合到統一的Emacs環境中，以減少上下文切換並提高效率。
